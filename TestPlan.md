# 📋 Test Plan: EsportHub Platform

**Мета:** Перевірка працездатності системи EsportHub (матчі, статистика, мапа турнірів).

| In-Scope (Що тестуємо) | Out-of-Scope (Не тестуємо) |

| Автооновлення рахунку (US 1.1) | Зовнішні API (карти, нотифікації) |

---

## 2. Стратегія тестування (Testing Strategy)

### Рівні тестування
* **Unit Testing:** Логіка розрахунку рахунку та фільтрації.
* **Integration Testing:** Взаємодія `Frontend ↔ Backend ↔ Database`.
* **System Testing:** Повний шлях користувача від авторизації до додавання в "Обране".
* **Manual Testing:** Перевірка відповідності макетам Figma.

### Види тестування
* **Functional:** Валідація оновлення даних та бізнес-правил.
* **API Testing:** Перевірка ендпоінтів `/matches`, `/players`, `/favorites`.
* **UI/UX:** Перевірка адаптивності таблиць та мапи.
* **Security:** Захист від некоректних даних (invalid input).

---

## 3. Ресурси та інструменти (Tools)
* **Інструменти:** Postman (API), PyTest (Unit), Chrome DevTools.
* **Bug Tracking:** Notion / Jira.
* **Середовище:** Chrome (120+), Node.js, MySQL Workbench.

---

## 4. Критерії прийняття (Acceptance Criteria)

### Entry Criteria (Початок)
* Код у гілці `develop`.
* База даних розгорнута.
* Наявні тестові дані для матчів та команд.

### Exit Criteria (Завершення)
* ✅ Виконано **100%** тест-кейсів.
* ✅ Відсутні баги пріоритету **Critical** та **High**.
* ✅ Pass Rate тестів не менше **90%**.
* ✅ Всі API повертають коректні відповіді.
* ✅ Система стабільно обробляє некоректні дані.

---

## 5. План виконання (Schedule)
1. **Тиждень 1:** API + БД тестування.
2. **Тиждень 2:** UI + валідація форм.
3. **Тиждень 3:** Інтеграційне тестування.

---

## 6. Очікувані результати (Deliverables)
